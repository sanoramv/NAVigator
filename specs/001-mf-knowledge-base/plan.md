# Implementation Plan: MutualLens — Indian Mutual Fund Knowledge Base

**Branch**: `001-mf-knowledge-base` | **Date**: 2026-05-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-mf-knowledge-base/spec.md`

## Summary

A local-first, single-page browser application for browsing, filtering, and comparing Indian
mutual fund Direct+Growth schemes. All ~2,000–3,000 eligible schemes are downloaded from two
free public APIs (AMFI NAVAll.txt for current NAV + metadata, MFAPI India for 5-year NAV
history), stored in IndexedDB, and never leave the user's machine. A virtual-scrolled table
with real-time filters and computed return/risk metrics gives the investor an immediately
useful shortlist tool — no server, no login, no cost.

**Critical scope decision (plan phase)**: Only Direct+Growth schemes are stored. AMFI
NAVAll.txt lists ~15,000 schemes; after filtering to name contains "Direct" AND "Growth",
~2,000–3,000 schemes remain. Plan Type and Option column/filters are therefore not needed
in the UI (every stored fund is already Direct and Growth).

## Technical Context

**Language/Version**: Vanilla JavaScript ES2022+ (no framework)
**Build Tool**: Vite 5.x
**Production Dependency**: `idb` 8.x (IndexedDB wrapper)
**Dev Dependencies**: `vite`, `gh-pages`
**Storage**: IndexedDB — 4 stores: `funds`, `nav_history`, `sync_meta`, `user_data`
**Testing**: Manual validation against spec acceptance scenarios; no automated test framework
**Target Platform**: Desktop browsers — Chrome 105+, Firefox 105+, Safari 16+, Edge 105+; deployed as GitHub Pages static site
**Project Type**: Static single-page web application
**Performance Goals**: Filter <200ms on 3,000 rows; virtual scroll at 60fps; cold page load <2s
**Constraints**: Zero server; zero API keys; offline after first sync; single prod dependency (idb); SVG-based NAV chart (no charting library)
**Scale/Scope**: ~2,000–3,000 Direct+Growth schemes; 5Y daily NAV history per fund (~1,300 data points per fund); single user, single browser

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Evidence |
|-----------|--------|----------|
| I. Investor-First | ✅ PASS | Direct+Growth filter pre-eliminates 80% of noise at the data layer; ≤15 default columns; opinionated filter defaults; no financial advice emitted |
| II. Local-Only Storage | ✅ PASS | IndexedDB only; no server component; offline after sync; sync is always user-triggered |
| III. Zero Lock-In | ✅ PASS | MIT license; AMFI + MFAPI are free and unauthenticated; CSV + JSON export; graceful API degradation |
| IV. Simple Distribution | ✅ PASS | `git clone && npm install && npm run dev` — three commands; `npm run deploy` for GitHub Pages; zero env vars or API keys |
| V. Honest About Data | ✅ PASS | Per-source sync timestamps in toolbar; formula + data-window tooltip on every computed metric; "N/A" for missing fields (AUM, expense ratio) |
| VI. Clarity Over Completeness | ✅ PASS | ≤15 default columns; Plan/Option filter controls removed (redundant — all funds are Direct+Growth); filters cover investor-useful dimensions only |
| VII. Performance | ✅ PASS | Filter runs in-memory on pre-sorted JS array; virtual scroll renders only visible rows; all sync runs in Web Worker off the UI thread |

*Post-Phase 1 re-check*: No design decision introduced during Phase 1 conflicts with any principle.

**Structure Decision**: Single project at repository root. No backend/frontend split. Source under
`src/` organised by responsibility layer: `db/` for persistence, `sync/` for data acquisition and
computation, `store/` for in-memory reactive state, `ui/` for rendering components, `config/` for
user-editable definitions. No build-time magic — every file is plain ES module JavaScript.
